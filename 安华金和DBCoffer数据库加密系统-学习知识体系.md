## 模块 2：产品定位

### 一句话定位

> **DBCoffer 数据库加密系统** 是一个面向 **政企、金融、运营商等需要数据库安全合规的行业** 的 **数据库透明加密产品**，帮助客户解决 **等保合规要求、敏感数据泄露风险、DBA等高权限用户越权访问** 三大核心痛点，相比 **Oracle TDE / 应用层加密** 的差异化优势在于 **完全应用透明、支持国密SM4、独立于数据库的密钥管理**。

### 目标客户画像

| 维度 | 说明 |
|------|------|
| 行业 | 政府、金融（银行/保险/证券）、运营商、医疗、教育、能源、军工 |
| 规模 | 中大企业（500人以上），数据库规模大（万级-亿级记录） |
| 典型角色 | 最终决策者：CIO/CISO/信息中心主任；技术决策者：DBA负责人/安全运维负责人；使用者：DBA/安全管理员 |
| 客户已有 IT 基础 | 主流数据库：Oracle、MySQL、PostgreSQL、达梦DM、人大金仓KingbaseES、Gbase、GaussDB |

### 核心痛点（客户视角）

1. **等保合规压力** — 等保2.0三级以上要求"对重要数据进行加密存储"，不满足则无法通过测评
2. **内鬼/越权泄露** — DBA、运维人员拥有数据库最高权限，可以直接查看明文敏感数据，数据泄露难以追溯
3. **应用改造成本大** — 传统应用层加密需要逐行修改代码，改造成本高、周期长、风险大

### 适用场景

1. **等保合规整改** — 满足等保2.0三级/四级"数据加密存储"要求，配合测评机构通过测评
2. **敏感数据保护** — 对身份证号、手机号、银行卡号、医疗记录等个人隐私信息进行加密保护
3. **信创国产化替代** — 替换Oracle TDE等进口加密方案，适配国产数据库和国密SM4算法
4. **防DBA越权** — 通过三权分立+应用绑定，限制DBA查看明文数据

### 费曼讲解（大白话给非技术决策者）

> DBCoffer 就像给您的数据库装了一把"专业保险箱锁"。
>
> 您数据库里的数据就像办公室里的文件。传统的做法是让每个员工（应用系统）自己给文件上锁——但这样每个员工都得学一遍开锁技能（改代码），麻烦得很。还有那个管钥匙的行政（DBA）随时可以打开任何抽屉看文件——万一他心术不正呢？
>
> DBCoffer 的做法是：在您的文件柜上加一把智能锁。所有员工放文件和取文件跟以前一样，不需要学开锁——对他们来说完全没变化。但这把锁的钥匙是独立管理的，连管文件柜的行政（DBA）都没有。谁、什么时候、从哪台电脑打开了哪个文件，全部有记录。
>
> 而且这把锁还通过了国家密码局的认证（国密SM4），在政府、银行、运营商这些对安全要求最高的地方都在用。部署起来也不复杂——就像给文件柜换把锁，里面文件的摆放顺序、使用习惯都不用变。

## 模块 12：术语解释

### 一、产品专有类

| 术语 | 全称 | 技术解释 | 大白话 |
|------|------|---------|--------|
| DBCoffer | DataBase Coffer | 安华金和的数据库加密产品品牌名 | 数据库保险箱，就是把数据库里的数据锁起来 |
| DBC安全服务器 | DBSec Server | 安华金和的核心加解密引擎，独立部署的硬件/软件服务器 | 专门负责给数据上锁和解锁的"保镖" |
| DBC安全代理 | DBSec Agent | 安装在数据库服务器上的插件，负责透明加解密的数据流拦截 | 安装在数据库上的一个小程序，自动识别哪些数据需要加密 |
| 三权分立 | Separation of Powers | 将DBA权限拆分为系统管理、安全管理、审计管理三个角色 | 管系统的不能管安全，管安全的不能看数据，管审计的只负责查记录——互相监督 |
| 应用绑定 | Application Binding | 将访问权限与应用系统进行绑定，防止绕过合法应用直接访问数据库 | 只有合法的App才能看数据，绕过App直接连数据库也看不到 |
| 密文索引 | Ciphertext Index | 对加密后的数据建立索引，保证密文状态下仍能高效检索 | 给锁在保险箱里的文件编个目录，不用一个个打开箱子就能找到你要的文件 |

### 二、协议/标准类

| 术语 | 全称 | 技术解释 | 大白话 |
|------|------|---------|--------|
| TDE | Transparent Data Encryption | 透明数据加密技术，在数据库存储层自动加解密，对应用透明 | 数据库自动帮你加解密，应用程序完全没感觉 |
| SM2/SM3/SM4 | 国密算法（SM2非对称/SM3杂凑/SM4对称） | 中国国家密码管理局发布的商用密码算法标准 | 中国的自有加密标准，比AES/RSA多了个"国家认证" |
| JDBC/ODBC | Java Database Connectivity / Open Database Connectivity | 数据库连接的标准接口 | 应用程序连数据库的"标准插头" |
| OLTP/OLAP | Online Transaction Processing / Online Analytical Processing | 联机事务处理（增删改查）/联机分析处理（报表分析） | 日常业务操作 vs 数据分析操作 |
