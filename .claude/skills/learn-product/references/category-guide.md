# 品类差异化适配指南

当 `/learn-product <产品名> [品类]` 指定了品类参数后，应用以下规则微调各模块输出。

---

## 品类 1：网络

**识别关键词**：交换机、路由器、防火墙、无线、SD-WAN、负载均衡、广域网优化、网络准入、上网行为管理

### 各模块调整

| 模块 | 调整规则 |
|------|---------|
| 技术架构 | 强调端口密度、吞吐量、交换容量、包转发率、组网协议支持 |
| 核心功能 TOP 5 | 侧重：性能、可靠性（冗余设计）、协议兼容、管理便捷性、扩展能力 |
| FAQ | 使用网络品类追问（兼容性、性能测试、替换割接、稳定性、多厂商互通） |
| 竞品矩阵 | 维度：吞吐量、时延、扩展性、运维复杂度、生态兼容、本地化支持 |
| 行业场景 | 侧重：多分支机构互联、数据中心组网、无线办公、远程接入 |

### 关键指标（售前必记）

- 吞吐量（Throughput）
- 时延（Latency）
- 并发连接数
- 包转发率（PPS）
- 端口密度/类型
- 冗余机制（堆叠/VRRP/双活）

---

## 品类 2：IT 基础设施

**识别关键词**：服务器、存储、超融合、虚拟化、备份、容灾、私有云、超融合一体机、SAN/NAS

### 各模块调整

| 模块 | 调整规则 |
|------|---------|
| 技术架构 | 强调计算规格、存储容量、IOPS、虚拟化支持、扩展方式 |
| 核心功能 TOP 5 | 侧重：性能/IOPS、扩容灵活性、数据保护、管理简化、生态开放 |
| FAQ | 使用 IT 基础设施品类追问（兼容性、扩容、迁移、售后、TCO） |
| 竞品矩阵 | 维度：性能、成本/TCO、扩容路径、生态兼容、本地服务 |
| 行业场景 | 侧重：数据中心整合、业务连续性、老旧设备替换、虚拟化转型 |

### 关键指标（售前必记）

- CPU/内存/存储配置
- IOPS / 吞吐量
- 最大节点数/扩展上限
- 数据保护机制（快照/副本/异地）
- 兼容的操作系统/虚拟化平台

---

## 品类 3：安全

**识别关键词**：WAF、态势感知、SOC、SIEM、终端安全、零信任、数据安全、DLP、堡垒机、漏洞扫描、蜜罐、威胁情报、身份安全、SASE

### 各模块调整

| 模块 | 调整规则 |
|------|---------|
| 技术架构 | 强调检测引擎位置、Agent 部署、策略下发机制、威胁情报来源 |
| 核心功能 TOP 5 | 侧重：威胁检测能力、响应速度、覆盖面、合规支持、误报控制 |
| FAQ | 使用安全品类追问（性能影响、合规认证、误报、终端 Agent、应急响应） |
| 竞品矩阵 | 维度：攻击覆盖、合规对标、性能损耗、检测率/误报率、威胁情报 |
| 行业场景 | 侧重：等保合规、护网行动、数据安全法合规、勒索病毒防护 |

### 关键指标（售前必记）

- 检测率 / 误报率
- 攻击覆盖范围（OWASP/ATT&CK 覆盖度）
- 性能损耗百分比
- 合规标准覆盖（等保/CISP/ISO 27001/PCI-DSS）
- 威胁情报更新频率
- 响应时间（MTTD/MTTR）

---

## 品类 4：云/企业软件

**识别关键词**：公有云、私有云、混合云、云迁移、云管平台、MSP、OA、ERP、CRM、BI、低代码、RPA、协同办公、HRM、SCM

### 各模块调整

| 模块 | 调整规则 |
|------|---------|
| 技术架构 | 强调部署方式（公有/私有/混合）、多租户、API 开放性、数据集成交互 |
| 核心功能 TOP 5 | 侧重：业务匹配度、集成能力、易用性、数据安全合规、扩展生态 |
| FAQ | 使用云/企业软件品类追问（数据安全、系统集成、实施周期、升级、ROI） |
| 竞品矩阵 | 维度：功能覆盖、API 生态、总拥有成本（TCO）、本地化适配、实施服务 |
| 行业场景 | 侧重：跨部门协同、数据打通、降本增效、数字化转型 |

### 关键指标（售前必记）

- 支持的集成方式（API/Webhook/数据库直连）
- 用户并发数/许可证模式
- 数据驻留/合规认证
- 实施周期（典型）
- 续费率/客户留存率（SaaS 类）

## 全品类通用售前话术模式

### SCQA 框架

```
S (Situation)   — 描述客户现状："咱们公司目前的..."
C (Complication) — 指出挑战/痛点："但面临的挑战是..."
Q (Question)    — 引出问题："怎么才能在保证 XX 的前提下解决 YY？"
A (Answer)      — 给出方案："[产品名] 通过 [功能/技术] 可以..."
```

### FAB 法则

```
F (Feature)  — 功能："这个产品具备 [功能]"
A (Advantage) — 优势："这意味它 [对比优势]"
B (Benefit)   — 价值："对您来说，最终能 [带来的价值]"
```

### SPIN 销售法

```
S (Situation) — 背景问题：了解客户现状
P (Problem)   — 难点问题：发现客户痛点
I (Implication) — 暗示问题：放大痛苦的影响
N (Need-payoff) — 需求-效益问题：展示解决后的价值
```
